This guide takes you from zero to a fully working dual-portal Next.js app with Supabase auth, route protection, and the Afiyah login UI live on Vercel. No hand-waving. Every command is exact.

What you'll have at the end

Two portals (Sisters B2C + Partners B2B) with email + OAuth auth, route protection, and the full Supabase schema live in production.

~15 min

Step 1

Prerequisites

You need these installed before starting.

01
Node.js 18.17+

Check: node --version — install from nodejs.org if needed

02
A Supabase account

Free at supabase.com — you need an email + organisation

03
A Vercel account

Free at vercel.com — connect to GitHub for one-click deploys

04
Git repo ready

Create a new repo at github.com — name it afiyah-infinity

Step 2

Create Supabase Project

01
Go to supabase.com → New Project

Name it afiyah-infinity · Choose a region close to Bangladesh or UAE · Set a strong database password (save it)

02
Copy your API keys

Settings → API → copy Project URL and anon public key. You'll paste these into .env.local in Step 5.

03
Enable Email Auth

Authentication → Providers → Email → Enable. Optionally disable "Confirm email" for faster local dev testing.

💡 The free Supabase tier gives you 500MB database, 1GB storage, 50K monthly active users, and 2GB bandwidth — plenty for MVP and early users.
Step 3

Run the Schema

Open Supabase → your project → SQL Editor → New query. Paste the entire contents of afiyah_supabase_schema.sql and click Run.

⚠️ Run order matters. The schema file is already ordered correctly — don't split it into separate queries. Run the whole file at once.

What the schema creates

ObjectCountPurpose
Tables20profiles, b2c/b2b profiles, zakat, hajj, wallets, transactions, marketplace, governance
RLS Policies16Users can only access their own rows — mahram-aware privacy enforced at DB level
Triggers7Auto-create profile on signup, auto-compute 2% Zakat pool on every transaction
Views3v_b2c_dashboard, v_b2b_dashboard, v_zakat_pool_summary
Extensions3uuid-ossp, pgcrypto, pgvector (for Niyyah AI RAG)
✅ After running, go to Table Editor — you should see all tables listed. The circles table will have 5 seed rows (Infinity Circle, Hajj Savers, etc.)

Step 5

Set Up Environment Variables

Copy .env.local.example to .env.local and fill in your values:

bash
cp .env.local.example .env.local

Then edit .env.local — the only two values you need right now are:

.env.local
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
NEXT_PUBLIC_SITE_URL=http://localhost:3000
⚠️ Never commit .env.local to git. Check that .gitignore includes .env.local (Next.js adds this automatically).
Step 6

Configure Supabase Auth

Set the Site URL and Redirect URLs

In Supabase → Authentication → URL Configuration:

01
Site URL

Set to http://localhost:3000 for now. Update to production URL after Vercel deploy.

02
Redirect URLs — add both:

http://localhost:3000/auth/callback
https://YOUR_VERCEL_URL.vercel.app/auth/callback

Configure Email Templates (optional but recommended)

Authentication → Email Templates → update the confirmation email to match Afiyah's brand tone. Replace the default Supabase copy with Afiyah messaging.

Step 8

Enable OAuth Providers

In Supabase → Authentication → Providers:

🔵 Google (B2C Sisters)

Enable Google provider → create a project at console.cloud.google.com → OAuth 2.0 credentials → paste Client ID + Secret into Supabase.

Required for B2C

🍎 Apple (B2C Sisters)

Enable Apple provider → requires Apple Developer account ($99/yr) → create App ID + Service ID → configure Sign in with Apple.

Optional for MVP

💼 LinkedIn (B2B Partners)

Enable LinkedIn OIDC provider → create app at developer.linkedin.com → OAuth 2.0 settings → Sign In with LinkedIn using OpenID Connect.

Required for B2B

📧 Email (Both portals)

Already enabled in Step 2. Works out of the box. For production: add a custom SMTP via Resend to avoid Supabase's 3 emails/hour limit.

Already enabled
Step 9

Test Locally

Start the dev server

npm run devCopy

Open http://localhost:3000/login — you should see the Afiyah login page with the portal toggle.

Test checklist

✓
Login page loads with Sisters Portal by default

Toggle switches between Sisters and Partners portals

✓
Create a B2C account with email

Check Supabase → Authentication → Users — you should see the new user appear

✓
Sign out and sign back in

Confirm the session persists across page refreshes

✓
Visit /dashboard directly while logged out

Middleware should redirect you to /login?portal=b2c

✓
Visit /partner directly while logged out

Middleware should redirect you to /login?portal=b2b

✓
Submit a B2B application (Step 1 → 2 → 3)

Check Supabase → Table Editor → b2b_profiles — row should appear with kyb_status = 'submitted'

Step 10

Deploy to Vercel

Push to GitHub

git add . && git commit -m "feat: auth system with dual portal login" && git pushCopy

Import on Vercel

01
vercel.com → New Project → Import Git Repository

Select your afiyah-infinity repo → Framework Preset: Next.js (auto-detected)

02
Add Environment Variables

In the Vercel deploy screen → Environment Variables → add all values from your .env.local file. Change NEXT_PUBLIC_SITE_URL to your Vercel URL.

03
Deploy

Click Deploy → Vercel builds and deploys in ~60 seconds.

04
Update Supabase redirect URLs

Authentication → URL Configuration → add your Vercel production URL to Redirect URLs: https://YOUR_APP.vercel.app/auth/callback

🚀 You're live. Both portals are deployed. Auth is working. The Supabase schema is running in production. Ready for Phase 2: building out the B2C Sisters dashboard.
